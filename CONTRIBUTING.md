# Contributing

Thank you for your interest in contributing to ListenWrite English.

## How to Contribute

1. Fork the repository.
2. Create a feature branch:

```bash
git checkout -b feature/your-feature-name
```

3. Install dependencies:

```bash
npm install
```

4. Start the development server:

```bash
npm run dev
```

5. Make your changes.
6. Run a production build:

```bash
npm run build
```

7. Open a pull request with a clear description.

## Contribution Guidelines

- Keep the app mobile-first.
- Keep the learner workflow simple and focused.
- Do not add authentication unless it is part of an approved roadmap item.
- Do not commit real environment variables, tokens, API keys, subtitle files, or generated build output.
- Keep UI text simple and easy to understand.
- Prefer focused pull requests over large unrelated changes.

## Reporting Issues

When opening an issue, please include:

- What you expected to happen.
- What actually happened.
- Browser and device information.
- Steps to reproduce the problem.
- Screenshots or screen recordings when helpful.

## Security

Do not report secrets or sensitive project details in public issues. If you find a security concern, open a minimal issue asking for a private contact path.
